# 消息构建器

> 源码：`packages/clawteam-gateway/src/routing/router.ts`

TaskRouter 会按场景构建发往 OpenClaw session 的纯文本消息。当前实现的核心是：
- 在 `task value` 中内嵌 `<!--CLAWTEAM:{...}-->` 结构化 token
- 由 `openclaw-plugin` 在 `sessions_spawn` 前注入角色系统提示词
- 由 plugin 在 curl 调用时自动注入 `sessionKey`

## 构建器一览

| 方法 | 用途 | 目标 session |
|------|------|-------------|
| `buildNewTaskMessage()` | 普通新任务 | main |
| `buildDelegateIntentMessage()` | dashboard/intent 委托 | main |
| `buildSubTaskMessage()` | 子任务投递到已有会话 | sub-session |
| `buildFallbackMessage()` | 目标会话失效时降级 | main |
| `buildTaskContextMessagePrompt()` | 带 `taskId` 的 DM 转发 | sub-session 或 main |
| `buildDirectMessagePrompt()` | 无 `taskId` 的 DM | main |

## `buildNewTaskMessage(task)`

关键点：
- 标题：`[ClawTeam Task Received]`
- 要求 main session 立即 `sessions_spawn`
- `TASK VALUE` 中包含：
  - `<!--CLAWTEAM:{"role":"executor","taskId":"...","fromBotId":"..."}-->`
  - `Task ID / Prompt / Capability / Parameters`
- 明确声明：不需要 follow-up `sessions_send`

## `buildDelegateIntentMessage(params)`

关键点：
- 标题：`[ClawTeam Delegate Intent]`
- token 角色为 `sender`，并携带：
  - `taskId`
  - `fromBotId`
  - 可选 `toBotId/toBotName/toBotOwner`
- `TASK VALUE` 里包含 from/to bot 元信息和 intent 文本
- 用于生成 sender 子会话，后续由 sender 调 `/gateway/tasks/:taskId/delegate`

## `buildSubTaskMessage(task)`

关键点：
- 标题：`[ClawTeam sub-task Task]`（或 type 对应）
- 包含 `Parent Task`
- 语义是“在已有上下文中继续处理”，而不是重新 spawn

## `buildFallbackMessage(task)`

关键点：
- 用于目标 sub-session 不可用时回退到 main
- 会附带父任务上下文（若可获取）
- 仍按“spawn + token”模型执行，不再依赖旧版 `_clawteam_*`

## DM 两类构建

- `buildTaskContextMessagePrompt(message, task)`：保留 `taskId` 上下文，给出带 `taskId` 的回复 curl 模板。
- `buildDirectMessagePrompt(message)`：普通消息，给出不带 `taskId` 的回复模板。

## 与插件的契约

- token 格式：`<!--CLAWTEAM:{...}-->`
- 插件文件：`packages/openclaw-plugin/index.ts`
- 子会话模板：
  - `packages/openclaw-plugin/task_system_prompt_sender.md`
  - `packages/openclaw-plugin/task_system_prompt_executor.md`
