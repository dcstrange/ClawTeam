# 方案 3：注册新工具 sessions_spawn_tracked

> ⚠️ Historical Notice: 本目录为历史方案调研，不代表当前线上实现。
> 当前实现请参考：`docs/Gateway/GATEWAY_TASK_MANAGEMENT.md`、`docs/Gateway/消息构建器.md`、`docs/task-operations/README.md`。

## 核心思路

通过 `registerTool` 注册一个新工具 `sessions_spawn_tracked`，它的 schema 中**显式增加 `clawteam_taskId` 字段**。LLM 可以直接以结构化参数传入 taskId，无需编码进 label。

执行逻辑：
1. 从参数中提取 `clawteam_taskId`
2. 将剩余参数传给 `spawnSubagentDirect`（通过 plugin-sdk 暴露的网关调用）
3. 拿到 `childSessionKey` 后调用 `track_session`
4. 返回与 `sessions_spawn` 相同格式的结果

## 关键约束（源码验证）

查看 `src/plugins/tools.ts:112`：

```typescript
// 插件工具与已有工具同名时，直接报错跳过
if (nameSet.has(tool.name) || existing.has(tool.name)) {
  log.error(`plugin tool name conflict (${entry.pluginId}): ${tool.name}`);
  continue;
}
```

**结论：不能注册名为 `sessions_spawn` 的插件工具来替换原版。** 只能使用新工具名。

## 优点

- ✅ `clawteam_taskId` 是 schema 的一等公民，LLM 直接感知
- ✅ 无需编码约定，参数清晰、类型安全
- ✅ 可以在工具 description 中说明 taskId 的语义
- ✅ 执行逻辑完全自控，可做任意前/后处理
- ✅ 不修改源码

## 缺点

- ⚠️ LLM 需要被引导使用 `sessions_spawn_tracked` 而非 `sessions_spawn`
- ⚠️ 需要通过 System Prompt 或 Skill 说明工具选择规则
- ⚠️ 需要访问 `spawnSubagentDirect` 或等效的内部 API

## sessions-spawn-tracked-tool.ts

```typescript
import { Type } from "@sinclair/typebox";
import type { OpenClawPluginToolContext } from "openclaw/plugin-sdk";

// 完全复用 sessions_spawn 的参数结构，只追加 clawteam_taskId
const SessionsSpawnTrackedSchema = Type.Object({
  // ── 新增字段 ────────────────────────────────────────────────────────────
  clawteam_taskId: Type.String({
    description: "ClawTeam task ID to associate with this spawned session (e.g. 'CLAWTEAM-12345')",
  }),

  // ── 原 sessions_spawn 全部参数 ──────────────────────────────────────────
  task:               Type.String(),
  label:              Type.Optional(Type.String()),
  agentId:            Type.Optional(Type.String()),
  model:              Type.Optional(Type.String()),
  thinking:           Type.Optional(Type.String()),
  runTimeoutSeconds:  Type.Optional(Type.Number({ minimum: 0 })),
  thread:             Type.Optional(Type.Boolean()),
  mode:               Type.Optional(Type.Union([Type.Literal("run"), Type.Literal("session")])),
  cleanup:            Type.Optional(Type.Union([Type.Literal("delete"), Type.Literal("keep")])),
});

export function createSessionsSpawnTrackedTool(
  ctx: OpenClawPluginToolContext,
  trackSession: (taskId: string, sessionKey: string) => Promise<void>,
) {
  return {
    name: "sessions_spawn_tracked",
    label: "Sessions (Tracked)",
    description: `
Spawn a sub-agent session AND associate it with a ClawTeam task ID.
Use this instead of sessions_spawn when you have a clawteam_taskId to track.
All other parameters are identical to sessions_spawn.
    `.trim(),
    parameters: SessionsSpawnTrackedSchema,

    execute: async (_toolCallId: string, args: unknown) => {
      const params = args as Record<string, unknown>;
      const clawteam_taskId = String(params.clawteam_taskId ?? "").trim();

      if (!clawteam_taskId) {
        return jsonResult({ status: "error", error: "clawteam_taskId is required" });
      }

      // 去掉 clawteam_taskId，剩余参数透传给内部 spawn 逻辑
      const { clawteam_taskId: _ignored, ...spawnParams } = params;

      // ── 调用内部 spawn 逻辑 ───────────────────────────────────────────
      // 方式 A（推荐）：通过 callGateway 调用 sessions_spawn tool
      // 方式 B：直接调用 spawnSubagentDirect（需要 plugin-sdk 导出）
      let result: any;
      try {
        result = await callSpawnViaGateway(spawnParams, ctx);
      } catch (err: any) {
        return jsonResult({ status: "error", error: err.message });
      }

      // ── spawn 成功后调用 track_session ───────────────────────────────
      if (result?.status === "accepted" && result?.childSessionKey) {
        try {
          await trackSession(clawteam_taskId, result.childSessionKey);
        } catch (err: any) {
          // track_session 失败不影响 spawn 结果
          console.error(`[ClawTeam] track_session failed: ${err.message}`);
        }
      }

      return jsonResult(result);
    },
  };
}

// ── 内部 spawn 调用的两种实现方式 ─────────────────────────────────────────

// 方式 A：通过 HTTP 调用 gateway（不依赖内部 API）
async function callSpawnViaGateway(
  params: Record<string, unknown>,
  ctx: OpenClawPluginToolContext,
): Promise<any> {
  // 通过 plugin-sdk 暴露的 callGateway 接口
  // 具体实现取决于 plugin-sdk 的导出内容
  const { callGateway } = await import("openclaw/plugin-sdk");
  const response = await callGateway({
    method: "tools.invoke",
    params: {
      toolName: "sessions_spawn",
      toolArgs: params,
      sessionKey: ctx.sessionKey,
    },
  });
  return response;
}

function jsonResult(details: unknown) {
  return { details };
}
```

## index.ts

```typescript
import type { OpenClawPluginApi } from "openclaw/plugin-sdk";
import { createSessionsSpawnTrackedTool } from "./sessions-spawn-tracked-tool.js";
import { trackSession } from "./tracker.js";

export default {
  id: "clawteam-session-tracker",
  name: "ClawTeam Session Tracker",

  register(api: OpenClawPluginApi) {
    const config = api.pluginConfig ?? {};
    if (config.enabled === false) return;

    const track = (taskId: string, sessionKey: string) =>
      trackSession(taskId, sessionKey, {
        apiEndpoint: config.apiEndpoint,
        apiKey: config.apiKey,
        logger: api.logger,
      });

    // 注册新工具（工厂函数形式，每个 agent 实例化时调用）
    api.registerTool((ctx) =>
      createSessionsSpawnTrackedTool(ctx, track)
    );

    api.logger.info("[ClawTeam] sessions_spawn_tracked tool registered");
  },
};
```

## LLM 使用示例

LLM 接收到的 tool schema 将包含 `clawteam_taskId` 字段：

```json
{
  "name": "sessions_spawn_tracked",
  "description": "Spawn a sub-agent session AND associate it with a ClawTeam task ID.",
  "parameters": {
    "type": "object",
    "required": ["clawteam_taskId", "task"],
    "properties": {
      "clawteam_taskId": {
        "type": "string",
        "description": "ClawTeam task ID (e.g. 'CLAWTEAM-12345')"
      },
      "task": { "type": "string" },
      "label": { "type": "string" },
      "mode": { "type": "string", "enum": ["run", "session"] },
      "thread": { "type": "boolean" },
      ...
    }
  }
}
```

LLM 调用：
```json
{
  "name": "sessions_spawn_tracked",
  "arguments": {
    "clawteam_taskId": "CLAWTEAM-12345",
    "task": "Analyze the codebase for security vulnerabilities",
    "label": "Security Analysis",
    "mode": "run"
  }
}
```

## 引导 LLM 使用新工具

方式一：在 Agent 的 System Prompt 中添加说明

```markdown
## Tool Selection Policy

When spawning a sub-agent session:
- If a ClawTeam task ID is available in context, use `sessions_spawn_tracked`
  and pass the ID as `clawteam_taskId`.
- Otherwise, use `sessions_spawn` as normal.
```

方式二：通过 Skill 文件注入规则（`SKILL.md`）

```markdown
---
name: clawteam-spawn-policy
---

# ClawTeam Spawn Policy

Whenever you need to spawn a sub-agent and a `clawteam_taskId` is provided,
use `sessions_spawn_tracked` instead of `sessions_spawn`.

Example:
- task: "analyze security"
- clawteam_taskId: "CLAWTEAM-12345"
→ Call `sessions_spawn_tracked({ clawteam_taskId: "CLAWTEAM-12345", task: "analyze security" })`
```

| 对比项 | 方案 1/2（Hook 编码） | 方案 3（新工具） |

| 需要内部 API | ❌ | ⚠️ 需验证 plugin-sdk 导出 |
