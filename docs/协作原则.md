# ClawTeam 协作原则（强制约定）

> 状态：Active  
> 更新日期：2026-03-25  
> 适用范围：Dashboard / API Server / Gateway / OpenClaw Plugin / Prompt 模板

## 1. 文档目标

本文件定义 ClawTeam 在多 Bot 任务协作中的强制约定。  
凡与本文件冲突的实现，默认视为缺陷（bug）而非“行为差异”。

## 2. 强制级别

- `P0（代码硬约束）`：已在 API/Gateway/Plugin 中实现校验或阻断。
- `P1（产品强约束）`：设计上必须成立，若当前实现未完全覆盖，应作为修复目标，不可当作可选行为。

## 3. 协作强制约定

### 3.1 审批链与人类介入代理

> 关注点：结果提交、审批入口、人类介入消息是否遵循“delegator 代理链”。

| ID | 约定（MUST） | 级别 | 代码锚点 |
|---|---|---|---|
| `CT-COLLAB-001` | **审批代理链强制**：执行者提交结果必须先进入 `pending_review`，由委托者 Bot 审批（`approve/reject`）。不得绕过委托者 Bot 让 Dashboard 直接审批。 | P0 | `packages/api/src/task-coordinator/completer.ts`（`submitResult/approve/reject`）；`packages/api/src/task-coordinator/routes/index.ts`（`/all/:taskId/approve|reject` 返回 403） |
| `CT-COLLAB-002` | **submit-result 仅可提交最终结论且必须绑定产物证据**：`result` 必须是最终交付，且 gateway 路径下必须带 `artifactNodeIds`（非空、任务关联）。中间进度/提问必须使用 DM 或 `need-human-input`。 | P0 | `packages/api/src/task-coordinator/routes/index.ts`（`INVALID_SUBMITTED_RESULT`）；`packages/clawteam-gateway/src/gateway/gateway-proxy.ts`（submit-result + `artifactNodeIds` 校验） |
| `CT-COLLAB-003` | **人类介入代理原则**：跨用户协作中，需要人类介入的信息应通过“自己的 delegator bot 代理链”传递，不应由外部 bot 直接驱动本用户 Dashboard 审批动作。 | P1 | `packages/api/src/task-coordinator/routes/index.ts`（Dashboard 直批禁用是已落地保护） |
| `CT-COLLAB-014` | **冲突必须自动纠偏**：当 delegator 在 `pending_review` 请求 `need-human-input` 时，系统应尝试自动回滚到可执行态再重试，避免“人为手工救火”。 | P0 | `packages/clawteam-gateway/src/gateway/gateway-proxy.ts`（pending_review -> reject -> retry need-human-input） |

### 3.2 身份与权限边界

> 关注点：谁可以调用什么接口、是否存在身份冒用或越权。

| ID | 约定（MUST） | 级别 | 代码锚点 |
|---|---|---|---|
| `CT-COLLAB-004` | **直接委托与子委托权限分离**：直接委托仅允许原始委托者在 `pending` 状态操作；子委托允许任务参与者（`fromBotId/toBotId`）发起。 | P0 | `packages/api/src/task-coordinator/routes/index.ts`（`/:taskId/delegate`） |
| `CT-COLLAB-006` | **身份一致性（fromBotId）**：代理层声明的 `fromBotId` 必须与本地 bot 身份一致，防止伪造委托者。 | P0 | `packages/clawteam-gateway/src/gateway/gateway-proxy.ts`（delegate fromBotId 校验）；`packages/openclaw-plugin/index.ts`（sender 身份不一致直接 block） |
| `CT-COLLAB-007` | **禁止自委托**：`toBotId` 不得等于本地 botId。 | P0 | `packages/clawteam-gateway/src/gateway/gateway-proxy.ts`（self-delegation 拒绝） |
| `CT-COLLAB-008` | **角色权限必须严格匹配**：`accept` 仅执行者；`approve/reject/cancel` 仅委托者；`complete` 默认委托者（执行者仅允许失败上报场景）；`reset` 仅执行者。 | P0 | `packages/api/src/task-coordinator/completer.ts` |

### 3.3 任务建模与委托上下文

> 关注点：父子任务关系、上下文透传是否完整。

| ID | 约定（MUST） | 级别 | 代码锚点 |
|---|---|---|---|
| `CT-COLLAB-005` | **子任务必须有父子关联与独立会话映射**：子委托必须生成独立 `subTaskId`，并设置 `parentTaskId` 指向父任务；子任务自身也必须维护 sender/executor 的 session 绑定（与父任务分离）。 | P0 | `packages/api/src/task-coordinator/routes/index.ts`（`subTaskPrompt` 分支创建子任务）；`packages/api/src/task-coordinator/routes/index.ts`（`/:taskId/track-session`） |
| `CT-COLLAB-015` | **预选执行者信息必须透传到 sender 子会话**：若任务由 dashboard/intent 预先选定 `toBotId`，sender 初始提示中必须显式包含目标执行者 ID/名称/Owner。 | P0 | `packages/openclaw-plugin/index.ts`（`TARGET_EXECUTOR_BLOCK` 注入） |

### 3.4 会话绑定、状态语义与恢复边界

> 关注点：session 状态与 task 状态解耦、nudge/recovery 不越界。

| ID | 约定（MUST） | 级别 | 代码锚点 |
|---|---|---|---|
| `CT-COLLAB-009` | **track-session 只绑定，不推进任务状态**：会话绑定不等于任务已 accept/已开始。 | P0 | `packages/clawteam-gateway/src/gateway/gateway-proxy.ts`（`/gateway/track-session`）；`packages/api/src/task-coordinator/routes/index.ts`（`/:taskId/track-session`） |
| `CT-COLLAB-010` | **track-session 的 role 不信任调用方**：服务端必须按调用者身份推断 `sender/executor`，请求体 role 仅用于告警。 | P0 | `packages/api/src/task-coordinator/routes/index.ts`（role mismatch warn + inferredRole） |
| `CT-COLLAB-011` | **`need-human-input` 可从 pending 进入等待态**：参与者可在 `pending/accepted/processing/waiting_for_input` 调用，且 pending→waiting_for_input 被视为“已开始执行路径”的状态推进。 | P0 | `packages/api/src/task-coordinator/completer.ts`（`waitForInput`） |
| `CT-COLLAB-012` | **Session 状态不等于 Task 状态**：`session=idle/completed` 可能与 `task=processing` 并存；任务真值以 Task 状态机为准。 | P0 | `packages/clawteam-gateway/src/recovery/stale-task-recovery-loop.ts`；`docs/Gateway/GATEWAY_TASK_MANAGEMENT.md` |
| `CT-COLLAB-013` | **Nudge 仅提醒，不改任务状态**：Recovery 对合法等待态（`waiting_for_input/pending_review`）应跳过；非 dead 的耗尽不应直接终结任务。 | P0 | `packages/clawteam-gateway/src/recovery/stale-task-recovery-loop.ts` |
| `CT-COLLAB-023` | **父任务完成门禁**：存在未终态子任务时，父任务不得被 `approve` 或 `complete(status=completed)` 直接终结；仅允许委托者显式 `force=true` 覆盖。 | P0 | `packages/api/src/task-coordinator/completer.ts`（`assertNoActiveChildTasks`） |

### 3.5 文件空间与共享边界

> 关注点：文件默认可见性、任务协作共享范围、团队发布链路。

| ID | 约定（MUST） | 级别 | 代码锚点 |
|---|---|---|---|
| `CT-COLLAB-016` | **默认私有原则**：文件创建默认落在 `bot_private/<botId>` 或 `task/<taskId>`，不得默认进入团队共享空间。 | P0 | `packages/api/src/file-service/routes.ts`（`normalizeScopeInput` + create/upload 校验） |
| `CT-COLLAB-017` | **任务共享最小授权**：`task/<taskId>` 仅对该任务参与者（delegator/executor/sub-delegate 链路）可见；非参与者必须拒绝。 | P0 | `packages/api/src/file-service/routes.ts`（`isTaskParticipant` + `assertWritableScope` + `hasAccess`） |
| `CT-COLLAB-018` | **团队空间仅发布产物**：`team_shared` 仅存放最终发布产物，中间态文件不得直接写入。 | P0 | `packages/api/src/file-service/routes.ts`（bot 写 team_shared 直接拒绝） |
| `CT-COLLAB-019` | **发布由代理链驱动**：任务产物发布到 `team_shared` 必须由 delegator bot（或 owner 代理）执行，不允许执行者绕过代理链直接发布。 | P0 | `packages/api/src/file-service/routes.ts`（`canPublishForTask` + `/publish`） |
| `CT-COLLAB-020` | **跨 bot 访问禁止旁路**：跨 bot 文件访问必须通过“本 bot 代理链 + ACL 判定”，禁止直接读取他人私有空间。 | P0 | `packages/api/src/file-service/routes.ts`（`hasAccess` ACL/继承/default） |
| `CT-COLLAB-021` | **Gateway 任务文件快捷路由必须强制 task scope**：`/gateway/tasks/:taskId/files/*` 仅允许操作与该 task 关联的节点，不可借此旁路访问其他 scope。 | P0 | `packages/clawteam-gateway/src/gateway/gateway-proxy.ts`（`withTaskScope` + `validateTaskLinkedNode`） |
| `CT-COLLAB-022` | **子任务产物父任务可见性保证**：子任务在审批通过后，系统必须把已批准产物镜像到父任务 `task` 空间，保证父任务交付可见且可继续协作。 | P0 | `packages/api/src/task-coordinator/completer.ts`（sub-task artifact mirror）; `tests/multibot/scenarios/test_subtask_artifact_sync.py` |

## 4. 术语约定

- `Delegator / Sender`：发起委托的 bot（`fromBotId`）。
- `Executor`：执行任务的 bot（`toBotId`）。
- `Participant`：`fromBotId` 或 `toBotId` 任一方。
- `Parent Task`：被子委托拆分的上级任务。
- `Sub Task`：由 `subTaskPrompt` 创建并绑定 `parentTaskId` 的子任务。
- `Session Key`：OpenClaw 会话标识（`agent:main:*`），用于会话绑定与恢复，不直接代表任务状态。

## 5. 必测回归清单（建议作为 CI / E2E 最小集）

1. 执行者 `submit-result` 后必须进入 `pending_review`，且仅委托者可 `approve/reject`。
2. Dashboard 直连 `/all/:taskId/approve|reject` 必须返回 403。
3. 直接委托（pending）与子委托（参与者）权限边界必须保持不变。
4. `fromBotId` 冒用必须在 gateway/plugin 被阻断。
5. `need-human-input` 在 `pending` 可成功进入 `waiting_for_input`。
6. `track-session` 不能替代 `accept`，且 role 由服务端推断。
7. 非 dead stale session 达到最大尝试后不能直接终结任务。

## 6. 变更治理

任何涉及以下模块的变更，在合并前必须回归本文件约定：

- `packages/api/src/task-coordinator/*`
- `packages/clawteam-gateway/src/gateway/*`
- `packages/clawteam-gateway/src/recovery/*`
- `packages/openclaw-plugin/*`

若实现与本文件冲突：

1. 以代码真实行为为准临时止血；
2. 立即补回归测试；
3. 在同一迭代修正文档与代码一致性；
4. 对 `CT-COLLAB-*` 规则编号进行变更记录（新增/废弃/替换）。
